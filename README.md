**Web Music Player (Self-Hosted)**
_**Overview**_

This project started as a simple class multimedia assignment aimed at building a basic web-based music player. What began as a small academic exercise will evolve into something much more ambitious — a self-hosted music streaming solution designed to eventually replace traditional platforms like Spotify.

As I continue building my home server, this project will grow into a fully-featured personal music system that gives me full control over my library, playback, and organization.

The long-term goal is to create a self-hosted music platform that:

Automatically detects and loads music from a specified directory
Organizes tracks intelligently (by artist, album, genre, etc.)
Supports playlists and queue management
Provides a clean, responsive web interface
Works seamlessly across devices on a local network (and possibly beyond)

_**Current Features**_
Basic web-based music player UI
Playlist display (static/manual for now)
Audio playback controls (play, pause, skip)
Simple navigation between songs

_**Planned Features**_

This project is actively evolving. Upcoming improvements include:

_1. Automatic Music Loading_
Scan a directory on the server for audio files
Dynamically update the library when new music is added

_2. Smart Organization_
Extract metadata (artist, album, genre)
Automatically categorize songs
Group tracks into albums and collections

_3. Playlist Support_
Create, edit, and delete playlists
Save playlists persistently
Drag-and-drop queue management

_4. Genre & Tagging System_
Assign genres automatically or manually
Filter and search music by tags

_5. Server Integration_
Integrate with a home server environment
Serve music over a local network
Potential remote access support

_6. Enhanced Player Features_
Shuffle and repeat modes
Audio progress tracking
Volume control improvements
